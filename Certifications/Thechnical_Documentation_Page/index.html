0. Introduction

SQL (Structured Query Language) is a powerful tool for managing and manipulating relational databases. Whether you're a database administrator, developer, or data analyst, understanding SQL is essential for working with structured data. This documentation will guide you through SQL fundamentals, database management, querying data, data modification, advanced topics, security, performance optimization, error handling, and troubleshooting.

1. SQL Basics

1.1. Basic SQL Query

At its core, SQL is all about querying data from databases. The most basic SQL statement is the SELECT statement, which retrieves data from one or more tables. Here's a breakdown of a basic SQL query:

SELECT column1, column2 FROM table WHERE condition;
  • SELECT: This keyword specifies the columns you want to retrieve in the result set. You can use * to select all columns.
  • FROM: Specifies the table or tables from which you want to retrieve data.
  • WHERE: Optional clause that allows you to filter the data based on a specified condition. The condition is used to include only the rows that meet specific criteria.

1.2. Common SQL Commands

SQL provides a set of powerful commands for interacting with databases. Here are some of the most common SQL commands:

  • SELECT: Used to retrieve data from one or more tables. Example:
SELECT first_name, last_name FROM employees;
  • INSERT: Adds new records to a table. Example:
INSERT INTO products (product_name, price) VALUES ('Laptop', 999.99);
  • UPDATE: Modifies existing records in a table. Example:
UPDATE customers SET email = 'newemail@example.com' WHERE customer_id = 123;
  • DELETE: Removes records from a table. Example:
DELETE FROM orders WHERE order_id = 456;
2. Database Management

2.1. Creating Databases

In SQL, a database is a structured collection of data organized into tables, views, and other objects. Before you can work with data, you need to create a database to store it. Here's how to create a database in SQL:

CREATE DATABASE database_name;
  • CREATE DATABASE: This SQL statement is used to create a new database. You specify the name of the database you want to create. The database_name should be unique within your database management system (DBMS).

2.2. Data Types

SQL supports various data types that define the kind of data a column can hold. Choosing the appropriate data type for each column is crucial for efficient data storage and retrieval. Here are some common data types in SQL:

  • INTEGER: Stores whole numbers.
  • VARCHAR(size): Variable-length character strings with a specified maximum size.
  • DATE: Stores date values (e.g., '2023-09-19').
  • DECIMAL(precision, scale): Exact numeric values with a specified number of digits and decimal places.
  • BOOLEAN: Stores true or false values.

When creating tables, you'll need to specify the data type for each column to ensure that the database correctly stores and interprets the data.

2.3. Creating Tables

Tables are the primary containers for storing data in a database. To create a table, you use the CREATE TABLE statement. Here's a basic example of creating a table:

CREATE TABLE employees ( employee_id INTEGER PRIMARY KEY, first_name VARCHAR(50), last_name VARCHAR(50), hire_date DATE );
  • CREATE TABLE: This statement creates a new table named "employees."
  • Inside the parentheses, you define the columns of the table along with their data types. In this example, we have columns for employee ID, first name, last name, and hire date.
  • The PRIMARY KEY constraint is used to ensure that the "employee_id" column contains unique values for each row. It serves as the table's primary key, which can be used to uniquely identify each employee.

2.4. Defining Relationships

In a relational database, tables are often related to one another. You can establish these relationships using foreign keys. A foreign key is a column or set of columns in one table that refers to the primary key of another table. This enforces referential integrity and ensures data consistency.

Here's an example of creating a table with a foreign key relationship:

CREATE TABLE orders ( order_id INTEGER PRIMARY KEY, customer_id INTEGER, order_date DATE, FOREIGN KEY (customer_id) REFERENCES customers (customer_id) );
  • In this example, the "orders" table has a "customer_id" column that references the "customer_id" column in the "customers" table. This relationship ensures that orders can only be associated with existing customers.
3. Querying Data

3.1. Retrieving Data

SQL's primary function is to retrieve data from databases. The foundation of data retrieval lies in the SELECT statement:

SELECT column1, column2 FROM table WHERE condition;
  • SELECT: This keyword specifies the columns you want in the result set. You can use * to select all columns.
  • FROM: Specifies the table or tables from which you want to retrieve data.
  • WHERE: An optional clause for filtering data based on specified conditions. It includes only the rows that meet the criteria.

3.2. Filtering, Sorting and Aggregating

  • Filtering: Use operators like =, <,>, etc., to filter data based on specific values or ranges.

Example:

SELECT product_name, price FROM products WHERE price < 50;
  • Sorting: Arrange the result set using the ORDER BY clause.

Example:

SELECT employee_name, salary FROM employees ORDER BY salary DESC;
  • Aggregating: SQL offers aggregate functions like COUNT, SUM, AVG, to perform calculations on grouped data.

Example:

SELECT department, AVG(salary) FROM employees GROUP BY department;

3.3. JOIN Clauses

When data is distributed across multiple tables, SQL allows you to combine them using JOIN clauses. Here are some common types:

  • INNER JOIN: Retrieves rows that have matching values in both tables.

Example:

SELECT orders.order_id, customers.customer_name FROM orders INNER JOIN customers ON orders.customer_id = customers.customer_id;
  • LEFT JOIN (or LEFT OUTER JOIN): Retrieves all rows from the left table and the matching rows from the right table. If there's no match, it still includes the left table's rows.

Example:

SELECT customers.customer_name, orders.order_date FROM customers LEFT JOIN orders ON customers.customer_id = orders.customer_id;
  • RIGHT JOIN (or RIGHT OUTER JOIN): Similar to LEFT JOIN but includes all rows from the right table.

Example:

SELECT customers.customer_name, orders.order_date FROM customers RIGHT JOIN orders ON customers.customer_id = orders.customer_id;
  • FULL JOIN (or FULL OUTER JOIN): Retrieves all rows when there is a match in either the left or the right table.

Example:

SELECT customers.customer_name, orders.order_date FROM customers FULL JOIN orders ON customers.customer_id = orders.customer_id;
4. Data Modification

4.1. Inserting Data

SQL provides the INSERT INTO statement to add new records to a table. Here's how to use it:

INSERT INTO table_name (column1, column2, ...) VALUES (value1, value2, ...);
  • INSERT INTO: This SQL command specifies the target table where you want to insert data.
  • Within parentheses, list the columns into which you want to insert data.
  • In the VALUES clause, provide the corresponding values for each column.

Example:

INSERT INTO products (product_name, price) VALUES ('Laptop', 999.99);

4.2. Updating Data

To modify existing records in a table, use the UPDATE statement:

UPDATE table_name SET column1 = new_value1, column2 = new_value2, ... WHERE condition;
  • UPDATE: This command identifies the table you want to update.
  • Use the SET clause to specify the columns to be modified along with their new values.
  • The optional WHERE clause filters the rows to be updated based on a condition.

Example:

UPDATE customers SET email = 'newemail@example.com' WHERE customer_id = 123";

4.3. Deleting Data

The DELETE FROM statement removes records from a table:

DELETE FROM table_name WHERE condition;
  • DELETE FROM: This SQL command specifies the table from which you want to delete records.
  • The optional WHERE clause determines which rows to delete based on a specified condition.

Example:

DELETE FROM orders WHERE order_id = 456;

It's crucial to exercise caution when using DELETE, as it permanently removes data from the database.

5. Advanced SQL

5.1. Subqueries

Subqueries, also known as nested queries, are a powerful feature of SQL. They allow you to embed one query within another. Subqueries are used for various purposes, such as data retrieval, filtering, and comparisons.

Example of a subquery for data retrieval:

SELECT product_name FROM products WHERE category_id IN (SELECT category_id FROM categories WHERE category_name = 'Electronics');

In this example, the subquery retrieves the category_id for the 'Electronics' category, and the main query retrieves product names belonging to that category.

5.2. Views

SQL views are virtual tables that store the results of a SELECT query. Views simplify complex queries and provide a way to present specific subsets of data to users without exposing the underlying table structure.

Example of creating a view:

CREATE VIEW customer_contacts AS SELECT customer_id, first_name, last_name, email FROM customers;

You can then query the "customer_contacts" view just like you would a table, even though it doesn't store data itself.
